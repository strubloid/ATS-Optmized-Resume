# AI Transferable Evidence

CurriculumOptimizer treats adjacent experience as a review aid, never as direct technology evidence. Deterministic matching runs first and identifies direct, equivalent, transferable, incomplete, or unsupported evidence. AI may explain an approved relationship and propose qualified wording, but it may not replace the technology named in `resume.md`.

## Safe workflow

1. Retrieve relevant source sections and bullets.
2. Classify evidence deterministically.
3. Ask an optional provider to explain the relationship using only approved evidence IDs.
4. Validate the structured response and reject unsupported claims.
5. Show the user the evidence, qualification, risk, and proposed wording.
6. Apply only to the generated job copy after confirmation.
7. Offer a separate reviewed edit for adding confirmed facts to the master resume.

Java experience may support backend concepts relevant to Node.js, but it never becomes Node.js experience. Transferable wording must retain the actual technology and explicitly qualify the relationship.
