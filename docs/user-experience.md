# User Experience

The UI should feel simple, calm, and confident.

## Main pages

- Dashboard
- Master Resume
- Companies
- Job Applications
- Generated CV Review
- Annotated CV Review
- Score Review
- Export Center
- Settings

## Review screens

- Show the generated CV in a document-like layout.
- Show comments beside or around the CV.
- Show score and breakdown nearby.
- Allow accept, reject, resolve, and edit actions.
- Support clean preview mode with no editor noise.

## Friendly labels

- Good fit
- Applicant Tracking ready
- Keywords well covered
- Strong storytelling
- Needs evidence
- Missing requirement
- Parser risk
- Unsupported claim
- Export ready
- Clean PDF ready

## UX rules

- Keep the master resume visible as the source of truth.
- Make unsupported requirements obvious.
- Show why a suggestion matters.
- Keep export actions easy to find.
